# 添加项目日志系统 - 收尾计划

## 摘要

继续之前已完成约 70% 的日志系统添加工作。本计划聚焦剩余 5 个文件的日志补全，其中 `models/vod_info.py` 和 `ui/cli_app.py` 是用户最初遇到的 "URL 为空" bug 的关键暴露点，必须补齐 WARNING 日志以形成完整的 bug 诊断链。

用户原话："先不修这个bug，给项目加上足够的日志，以后除了问题，我就带着日志来找你" —— 因此本计划只加日志，不修 bug。

- `parsers/direct_parser.py`、`js_variable_parser.py`、`web_sniff_parser.py`、`json_api_parser.py` - 均 except 用 `exc_info=True`

未完成（5 个，本计划范围）：

3. `player/video_player.py` - 完全无日志，仍有诊断 print

**关键发现**：设计期望的 bug 诊断链 `vod_info.parse_play_list WARNING → cli_app._play_episode WARNING → parser_manager.parse ERROR` 目前是断的（中间环节缺失），用户运行程序后无法从日志看到 bug 根因。

## 设计原则（沿用既有约定）

- **print vs logger 划分**：UI 交互输出保留 print（如菜单、提示、分隔线）；诊断信息用 logger
- **模块级 logger**：每个文件顶部 `import logging` + `logger = logging.getLogger(__name__)`
- **异常记录**：所有 except 块使用 `logger.error(..., exc_info=True)` 记录完整堆栈
- **日志消息语言**：中文
- **不过度日志**：不记录循环每次迭代的完整数据，只记录关键节点

### 1. `parsers/spider_parse_parser.py`（补全末尾 except 块）

**文件**：`c:\MyTools\TvBox_WZ\parsers\spider_parse_parser.py`

**改动**：将末尾 2 个 except 块（约第 155-158 行）从单纯 return 错误字典改为先 `logger.error(..., exc_info=True)` 再 return，与 `js_variable_parser.py` 等其他解析器保持一致。

```python
# 改动前
except json.JSONDecodeError as e:
    return {"url": "", "success": False, "error": f"Spider 返回 JSON 解析失败: {str(e)[:100]}"}
except Exception as e:
    return {"url": "", "success": False, "error": f"Spider 解析失败: {str(e)[:100]}"}

# 改动后
except json.JSONDecodeError as e:
    logger.error("Spider 返回 JSON 解析失败: %s", e, exc_info=True)
    return {"url": "", "success": False, "error": f"Spider 返回 JSON 解析失败: {str(e)[:100]}"}
except Exception as e:
    logger.error("Spider 解析失败: %s", e, exc_info=True)
    return {"url": "", "success": False, "error": f"Spider 解析失败: {str(e)[:100]}"}
```

**原因**：与其他 4 个解析器保持一致；当前是唯一未记录异常堆栈的解析器。
